iOS Fastlane --自动化打包上传到蒲公英、fir

发表于2020-05-11|更新于2020-05-11|Fastlane
|阅读量:

参考： iOS使用fastlane实现自动化打包

安装 homebrew

/bin/zsh -c "$(curl -fsSL https://gitee.com/cunkai/HomebrewCN/raw/master/Homebrew.sh)"

查看homebrew版本 brew -v

更新 brew update

本地软件库列表 brew ls

检查 ruby版本，以及是否安装 Xcode 命令行工具

ruby -v 查看是否安装ruby

xcode-select --install 安装Xcode工具

安装 fastlane

sudo gem install fastlane

之后 cd 到项目根目录中

fastlane init 之后会出现如下图的4个选项：

自动截取APP的屏幕
发布bate版用于TestFlight
发布到appStore
自定义任务

肯定是选择4，选择完成后项目目录中多出了这些文件

文件配置

配置 Appfile (这里填app的bundleId)
配置 Fastfile

注意： APP_PGYER_API_KEY、 APP_PGYER_USER_KEY、 APP_FIR_API_TOKEN分别要去蒲公英和fir的官网上申请的。
在进行打包前我们还要安装蒲公英和fir的插件
fastlane add_plugin pgyer、 fastlane add_plugin firim 、fastlane add_plugin fir_cli

执行任务

在 Fastfile 配置文件中我们可以看到有四个任务，分别是：iosDebug_pgy、 iosDebug_fir 、iosTestFlight、 iosRelease ，每个任务里都有不同的配置，如果想要自行配置可在文件中进行修改，我们只需要在项目根目录下执行 fastline iosDebug_pgy 、fastlane iosDebug_fir..等对应的任务名即可。这样就成功了

遇到的问题

当我在打Release包的时候，遇到了下面这些错误

看到这些报错我以为是文件导入方式不对，但改了之后也是报错，于是我仔细看了第一句错误Couldn't find specified configuration 'Release'，我点开了Edit Shceme，然后我发现Build Configuration 选项中只有Debug没有Release ，在下面添加了Release 模式，重新运行即可打包成功。
